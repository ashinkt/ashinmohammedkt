🎓 What This Scan Taught Us

🕷️

Web Crawling

The scanner fetched the target URL and used BeautifulSoup to parse the HTML DOM, collecting every <a href> tag — exactly how search engines and security tools map a site's attack surface.

🔌

HTTP Status Codes

200 = OK  ·  301/302 = Redirect  ·  404 = Broken  ·  500 = Server Error. These codes are the primary signal used to classify link risk in automated security assessments.

⚡

Concurrent Threading

Checking {{ total }} links sequentially could take minutes. A ThreadPoolExecutor runs 20 threads in parallel to check every link simultaneously — the same technique used in real-world penetration testing tools like Nikto and Gobuster.

🛡️

Responsible Disclosure

Uses lightweight HEAD requests (no body download) to minimise server load. Falls back to GET only when the server rejects HEAD. This mirrors responsible disclosure principles — verify risk without causing harm.

⚠️

What Makes a Link "Risky"?

Broken (4xx/5xx): Dead links open expired domain hijacking opportunities. Redirects (3xx): Open redirects are a classic phishing vector — a trusted URL silently forwards users to a malicious site. External links introduce third-party trust dependencies attackers can exploit.

🏭

Supply Chain Connection

Supply chain portals link suppliers, logistics APIs, and payment gateways via hyperlinks. A single compromised redirect can lead to credential theft; a hijacked broken-link domain can serve fake invoices. IBM reports 62% of breaches involve a third-party supplier — link integrity is the first defence layer.

🔬

Zero-Day Scanner vs Basic Checker

A basic link checker only reports whether a link works. This Zero-Day Scanner goes further: it risk-classifies every link (broken / redirect / external), surfaces hidden vulnerabilities site owners are unaware of — acting like a zero-day probe for link-layer weaknesses before an attacker finds them first.

🧑‍💻

Ethical Hacking Workflow

This scan sits in Phase 2 — Reconnaissance of the pentest lifecycle: Authorise → Recon (here) → Analyse → Exploit (in scope) → Report. The colour-coded results feed directly into Phase 5 — a structured report with severity ratings and remediation advice.
